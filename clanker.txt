MinimalistHUD project notes for future Codex agents

Project structure
- `src_vanilla/` is the full exported vanilla ActionScript source dump.
- `src/` is the modded source set used for rebuilding/importing. It may contain only the files intentionally overridden.
- When a needed class exists only in `src_vanilla/`, copy it into the matching path under `src/` before modifying it.

Editing rules that match how this project is being worked on
- Preserve the watch gameplay widgets unless specifically asked to remove them.
- Prefer suppressing HUD elements at widget/controller entry points like `onSetData()` instead of ripping out embedded assets.
- Keep changes scoped to `src/`. Treat `src_vanilla/` as reference only.

Notes on bluff/lure strip
- The bottom bluff/lure widget is the agency selector strip, not the circular agency meter.
- `AgencySelectorWidget` owns the frame, lines, icon, and chunk count.
- `AgencySelectorWidgetEntry` owns each individual prompt glyph, flare effect, and per-entry cost/label layout.
- Current mod behavior hides the frame and chunk count, hides entry labels and costs, and keeps only the prompt glyphs plus flares when enabled.

Notes on standard button prompts
- The shared interaction prompt pill/glyph renderer is `ButtonPromptWidget`.
- Vanilla behavior uses `ButtonPromptImage` instances with optional per-prompt inversion and platform-specific scaling.
- Current mod behavior forces prompt images through the inverted color matrix so the buttons render dark with light glyphs, and applies a `0.8` multiplier to the normal prompt icon scale.
- `CloseCombatPromptsWidget` is separate and does not inherit those shared prompt changes automatically.
- Current mod behavior also forces its `ButtonPromptImage` instances through the inverted color matrix and applies the same `0.8` scale multiplier there.
- `NPCCombatPromptsWidget` is also separate and directly instantiates its own combat prompt images.
- Current mod behavior also forces those combat prompt images through the inverted color matrix and applies the same `0.8` scale multiplier there.

Notes on takedown/throw prompt circles
- Silent takedown / melee finisher prompt background is driven by `NPCCombatPromptsWidget.promptBackground()` through `button_mc.anim_mc`.
- Close-combat prompt ring/outline behavior exposed in script is in `CloseCombatPromptsWidget` via `combo_mc` / `combo_mc.outline_mc`.
- If circles still remain after testing, the remainder is likely baked into the prompt symbol art or `ButtonPromptImage` rendering rather than a separate controller toggle.

Notes on direction arrows
- `AttentionGainWidget` was already overridden and now hard-hides the wedge-based attention indicators in `onSetData()`.
- Remaining red directional combat arrows were traced to the damagedirection controllers, not the attentiongain widget.
- `OffscreenEnemyDirectionWidget` colors a container with `MenuConstantsKnt.COLOR_HUD_DANGER_HIGH` and shows per-enemy indicators when `newCombatant` is true.
- `OffscreenMeleeDirectionWidget` drives offscreen melee/parry indicators separately.
- Current mod behavior ports both damagedirection classes into `src/` and keeps their vanilla object/state lifecycle intact while forcing the rendered indicators hidden. This is safer than fully short-circuiting `onSetData()` if combat stability depends on normal widget bookkeeping.

Notes on objectives widget
- The top-left mission/objectives list is controlled by `ObjectivesWidget`.
- `ObjectivesWidget` already had watch-state handling through `IsInQLens` / `m_isAimingWatch`.
- Current mod behavior keeps the objectives list hidden unless the watch/QLens is being aimed, and scales the whole widget to `0.85`.
- The centered tutorial-style banner is not the same widget; tutorial center-banner notifications also flow through `ObjectivesNotificationItemWidget` using `ObjectivesData.TYPE_TUTORIAL`.
- Current mod behavior suppresses only the `TYPE_TUTORIAL` branch there, leaving item, challenge, mosaic, and collectible notifications intact.

Notes on grenade indicator
- The flashbang / grenade warning indicator is driven by `knt.hud.breadcrumbs.GrenadeBreadcrumb`.
- This controller owns the pulse icon plus directional arrow for smoke, electric, explosive, and flash grenade breadcrumbs.
- Current mod behavior ports `GrenadeBreadcrumb` into `src/` and forces the widget hidden for all activate/deactivate and edge-angle paths.

Notes on tutorial widgets
- The side-of-screen tutorial info card is `TutorialHintWidget`.
- The stacked pop-up style tutorial notifier is `TutorialSimplePopUpWidget`.
- Earlier suppression that only killed animations was not strong enough.
- Current mod behavior also hides the widget roots/containers so reused state cannot bring them back on-screen.

Notes on focus gain popup
- The lightning-bolt `+N` popup for gained focus/instinct energy is driven by `knt.hud.notifications.AgencyNotificationWidget`.
- Vanilla behavior compares `agencyData.currentAgency` against `m_previousValue` and shows a popup when the value increases.
- Current mod behavior ports `AgencyNotificationWidget` into `src/` and suppresses the popup in `onSetData()` while still updating `m_previousValue` so repeated gains do not accumulate stale deltas.

Notes on NPC watcher indicators
- The white overhead watcher dot/icon is driven by `NpcStateWidget` via `state_mc.watcher_mc`.
- The influenced-by-watcher icon is also driven there via `state_mc.watcher_influenced_mc`.
- Vanilla behavior already gated `INFLUENCED_BY_WATCHER` on `m_isInWatchView`, but plain `WATCHER` / `WATCHER_INFLUENCING` did not require watch view.
- Current mod behavior ports `NpcStateWidget` into `src/` and short-circuits the widget entirely unless `m_isInWatchView` is true, so all NPC overhead state indicators from this widget only appear while using Q Lens/watch view.

Notes on weapon ammo bar
- The redundant right-side bullet strip is `WeaponWidgetView.ammo_bar`, controlled by `knt.hud.weapons.WeaponWidget`.
- Primary ammo bar activation happens when a weapon loads:
  `m_view.ammo_bar.visible = true`
  `drawAmmoBarSegments(data.nMaxAmount, 142, 90, 1)`
- Segment visibility is updated through `m_ammoBarSegments[...]` based on `nRemainingAmount`.
- Related supporting visuals are `gradient_lines_mc`, `ammo_icon_container_mc`, `ammo_info_mc`, and for gadget/resource weapons `special_ammo_bar`.
- If removing only the redundant bullet strip, `ammo_bar` is the first target. Be careful not to accidentally remove the numeric ammo text in `ammo_info_mc`.
- Current mod behavior gates the primary ammo bar off in `WeaponWidget` with a `SHOW_AMMO_BAR` constant, so the numeric ammo digits remain but the curved right-side primary ammo bar is no longer shown or updated.
- The leftover arc/separator visuals next to the removed ammo bar were `gradient_lines_mc`; current mod behavior keeps those hidden as well.

Notes on weapon reticle
- `WeaponReticleWidget` can appear to lose the idle aiming reticle if `ironsight_inner_mc` is left at a low/default alpha while the widened firing state still animates it back on.
- Current mod behavior explicitly restores `m_reticle.ironsight_mc.ironsight_inner_mc.alpha = 1` during reticle setup and when `shouldShowReticle(true, ...)` is called.

Testing notes
- The agent did not run the game. Rebuild/import and verify in-game after changes.
- If a HUD element still appears, search `src_vanilla/` for the controller first, then port only the needed file into `src/`.
